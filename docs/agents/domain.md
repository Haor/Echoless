# Echoless 领域文档

Echoless 使用单一上下文的领域文档结构。

进行架构或领域层变更前，读取：

- `CONTEXT.md`（如果存在）；
- `docs/adr/` 下适用于当前改动的架构决策记录。

这些文件尚不存在时直接继续，不需要警告。只有真实领域决策需要持久记录时才按需创建。

## 目录结构

```text
echoless/
├── CONTEXT.md
└── docs/
    └── adr/
```

`CONTEXT.md` 用于定义产品边界、核心音频概念、共享词汇、重要不变量和主要子系统关系。

架构决策记录用于保存无法仅通过代码可靠还原其理由的决策。

代码、Issue、计划和面向用户的技术文档应使用领域文档中定义的词汇。如果拟议工作与已有
架构决策记录冲突，必须在实现前明确指出冲突。
